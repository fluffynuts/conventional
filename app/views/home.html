Congratulations!

You have a simple Angular App running, using RequireJS and a convention-over-configuration strategy.

What can you do from here? You can add functionality via this set of steps per route:

  • Edit config.js and add the route you're interested in (eg 'foo')
  • Create a controller javascript file under js/controllers with the same name as your route (ie, in the example above, js/controllers/foo.js). You can use __shell.js as a starting point.
  • Don't forget to add your view under views, with the same name (again, views/foo.html)
  • Now you can flesh out your functionality and bound fields in your controller and set up your view to use it
